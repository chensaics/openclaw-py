# M2 双端能力基线与回归清单（Flet + Flutter）

> 适用里程碑：M2 双轨客户端  
> 目标：确保 Flet 稳定线与 Flutter 增强线共享同一 Gateway 能力语义

## 0) 自动化入口（已实现）

- [x] 已提供命令：`pyclaw ops m2-baseline`
- [x] 自动化覆盖子集：gateway 可见性、`skills run` 成功链路、`skills run` 失败阻断链路
- [x] 输出结构化结果：`decision`、`failed_required`、`manual_checks_pending`

示例：

```bash
pyclaw ops m2-baseline --json
```

## 1) 核心能力基线（必须一致）

| 能力域 | Flet | Flutter | 验收标准 |
|---|---|---|---|
| Gateway 连接与重连 | 必须 | 必须 | 断线重连成功，错误态可见 |
| 会话列表/切换/删除 | 必须 | 必须 | 同一数据集行为一致 |
| 聊天发送与流式回显 | 必须 | 必须 | 首 token 时延与终态一致 |
| 计划（Plan）与任务状态 | 必须 | 必须 | 状态机字段语义一致 |
| Cron 列表与启停 | 必须 | 必须 | 控制动作与反馈一致 |
| 系统状态（doctor/health） | 必须 | 必须 | 关键指标字段一致 |
| 日志查看与过滤 | 必须 | 必须 | 过滤结果一致 |
| 配置查看/编辑 | 必须 | 必须 | 保存后 gateway 生效一致 |
| Skills 列表与运行入口 | 必须 | 必须 | 能展示并触发同一 skill key |

## 2) 回归用例清单（每次版本必跑）

### 2.1 连接与会话

- [ ] 启动后连接 gateway 成功（含 token/password 两种路径）
- [ ] 手动断网后自动重连并恢复 UI 状态
- [ ] 会话创建、切换、删除、重命名成功

### 2.2 聊天主链路

- [ ] 发送普通文本并流式返回
- [ ] 中断流式响应后无残留占位
- [ ] 编辑并重发历史消息成功

### 2.3 运维链路

- [ ] `status/health` 关键字段可见
- [ ] logs 过滤（level + text）一致
- [ ] cron 启停与下一次执行时间更新

### 2.4 技能链路

- [ ] skills 列表展示（本地 + 已安装）
- [ ] 可触发 `pyclaw skills run` 路径并展示结构化结果
- [ ] 失败时展示 blocking_items 与建议动作

## 4) 退出条件（M2 DoD）

- 基线表中“必须”项全部通过。
- 回归用例通过率 >= 95%。
