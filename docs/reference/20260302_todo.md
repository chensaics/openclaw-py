  - `test_docs_parity_phase43.py` — 文档-命令契约
  - `test_naming_parity_phase49.py` — 命名一致性
- 新增 RPC 方法或 CLI 命令时，必须同步更新 `docs/api-reference.md` 和 PROGRESS.md，否则 CI 失败。

### 8.2 季度手动复核流程

每季度（1/4/7/10 月第一周）执行一次：

1. **查看 TS upstream CHANGELOG / Releases**  
   `https://github.com/openclaw/openclaw/releases` — 检查上一季度新增的 RPC 方法、CLI 命令、通道、提供商。
2. **对比 RPC 方法清单**  
   将 upstream 的 `docs/api-reference.md`（或等价）与本项目 `docs/api-reference.md` 做 diff，记录新增/变更项。
3. **评估优先级**  
   按 P0-P3 分类新增差距项，更新本文档 `七、优先级总结` 表。
4. **创建执行计划**  
   对 P0/P1 项立即排期，P2/P3 项按资源情况排入后续 Phase。
5. **更新 PROGRESS.md**  
   在功能对齐状态一节追加季度复核记录。
